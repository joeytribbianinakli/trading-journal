JSON / CSV Backup & Restore
Export JSON
Full export of all trades, images, sessions, settings and preferences. Keep this as your portable data file — import it on any device or browser.
Export CSV
Download trade data as a spreadsheet. Useful for analysis in Excel or Google Sheets. Does not include images or settings.
Import (JSON or CSV)
Upload a JSON data file or CSV — format is detected automatically. JSON restores everything; CSV imports trades only. Duplicates are skipped.

Repository Setup
Create a private GitHub repo (e.g. my-trading-journal), then generate a Personal Access Token at github.com/settings/tokens with repo scope. Your token is stored only in this browser.
Push to GitHub
Manually push your full journal (trades, images, sessions, settings) to your GitHub repo right now.
Pull from GitHub
Restore your journal from GitHub. Use this on a new browser or device. Existing data is merged (no duplicates).
Last sync

Deploy to GitHub Pages
⚠ Use a separate public repo for Pages. GitHub Pages requires a public repo on free accounts. Your data repo must stay private — it contains your full trade history. Create a second repo (e.g. trading-journal-app) that holds only this HTML file. Your GitHub sync credentials still point at your private data repo — nothing changes there.
Steps to deploy:
  1. Create a new public repo on GitHub (e.g. trading-journal-app) — not your data repo
  2. Add this index.html file to your repo root
  3. Download the workflow file below and commit it to .github/workflows/deploy.yml
  4. In that repo's Settings → Pages, set source to GitHub Actions
  5. Push — your journal will be live at https://<user>.github.io/trading-journal-app
  6. On any new device, open the URL, go to Backup & Restore, enter your private data repo credentials and hit Pull
Backup & Snapshots
Manual Backup
Push a timestamped snapshot to bkp/ in your GitHub repo right now. File name includes manual-backup so it's easy to tell apart from auto-snapshots. Useful before large changes.
Restore Latest Backup
Replace your current data with the most recent snapshot in bkp/. You can also pick a specific one from the list below. This overwrites your current data.
All Snapshots — bkp/ folder
manual-backup · session-snapshot · clear-all-snapshot
Snapshots are saved here manually or automatically before a session delete / clear-all. File names distinguish their origin: manual-backup-…, session-snapshot-…, clear-all-snapshot-…. Use Restore to apply directly, or Download and import via JSON/CSV above.
Cloudinary — Image Hosting Not configured
When configured, screenshots are uploaded to Cloudinary (free tier: 25 GB) instead of being stored as base64. Your tj_data.json stays tiny — only image URLs are saved, not the image data itself.

Sign up free at cloudinary.com, then go to Settings → Upload → Upload presets and create an unsigned preset. Copy your Cloud Name from the dashboard.

⚠ Required for screenshot deletion: In your upload preset settings, under Upload Control, set Return delete token to Yes. Without this, removed screenshots will not be deleted from Cloudinary.
How it works: When you paste or drop a screenshot, it's uploaded instantly to Cloudinary. The image URL is stored in your trade. Your GitHub backup stays lean — no base64 blobs. Existing trades with base64 images continue to work normally.
Danger Zone
Clear all data
Permanently delete every trade and image. Export a backup above first! If GitHub is configured, a snapshot is auto-saved before deletion.
